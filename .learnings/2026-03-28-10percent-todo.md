## TODO1: ターゲット1人・訴求1本への絞り込みルール

### 新ルール（適用開始: 2026-03-28）
デザイン/コピー提案を出す前に、**必ず以下の絞り込み案を最初に作成**する：

| 項目 | 内容 |
|------|------|
| **ターゲット** | 具体的な1人（例：「小学3年生の母・40代・働くママ」） |
| **訴求軸** | 1本のみ（例：「子供の自主性を育てる」） |
| **シーン** | その人が何を困っているかの具体的シーン |

### なぜこれが必要か
- 広く浅い提案 → 採用されにくい
- 狭く深い提案 → 共感されやすく、修正回数減少

- [ ] サムネイル作成依頼受注時
- [ ] コピー案発散前
- [ ] キャンペーンビジュアル企画時

## TODO2: failedセッションの共通点分析と再発防止

### 共通点: claude_code 使用時のAPIキーエラー

**エラーログ（2026-03-11-sales-lead-audit.md）:**
```
assistant: 95件も！ランキングサイト系除外してclaude_codeで一気にバッチ処理するよ🌹🔥
assistant: claude_codeがAPIキーエラー😅 自分でバッチ処理スクリプトで一気にやるよ🌹🔥
```

**発生条件:**
1. `claude_code` ツールを使用しようとした際
2. バッチ処理・大量処理を依頼した場合
3. APIキー認証エラーでセッションが失敗

### 再発防止の送り方メモ

**✅ 推奨する依頼パターン:**
```
# ❌ 避ける（失敗しやすい）
- claude_code で95件のバッチ処理
- 一気に大量のファイル生成

# ✅ 推奨（安定する）
- 少量ずつ（5-10件単位）に分割して依頼
- 標準的なスキルツール（exec/read/edit）を優先
- claude_code は単発・複雑なコーディングタスクのみに限定
```

**代案ツール:**
| 用途 | 代替ツール |
|------|-----------|
| バッチ処理 | exec + シェルスクリプト |
| ファイル生成 | write/edit |
| Web検索 | web_search / web_fetch |
| 画像生成 | image_generate |
