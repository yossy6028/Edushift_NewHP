# Copy Ideation Obsidian

商品・企画コピーのアイデアを求められたら、以下を実行する。

1. Obsidianのノートから素材を取る
- `obsidian`スキルに従って、ノートタイトル「AIを使って考える全技術」および著者名「石井力重」で検索する。
- 該当ノート内の「アイデア発散プロンプト」を複数抽出する。
- ユーザーの依頼に最も合うプロンプトを1つ以上選ぶ。

2. 選んだプロンプトを明示して発散する
- 回答内で「どの発散プロンプトを使ったか」を短く示す。
- 1回の回答で最低10案を出す（依頼が少数指定ならそれに従う）。
- 似た表現を避け、切り口を分散する（ベネフィット、感情、対比、具体シーン、数字、比喩など）。

3. 出力フォーマット
- セクションA: 使用プロンプト（1〜2行）
- セクションB: コピー案（箇条書き）
- セクションC: 推し3案と理由（各1行）
- 必要なら「別プロンプトでもう1ラウンド」を提案する。

4. 運用ルール
- ノートが見つからない場合は、その旨を伝えたうえで通常のコピー発散を行い、後でノート位置の確認を依頼する。
- 既存案のリライト依頼では、原案の意図を保持しつつ別角度の案を混ぜる。
- 薬機法・法務・誇大表現リスクがある業種では、断定表現を避けた案も混ぜる。
