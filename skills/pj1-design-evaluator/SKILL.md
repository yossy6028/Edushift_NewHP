# PJ1 Design Evaluator

PJ1パイプラインにおけるデザイン評価タスクの標準化スキル。

## 評価対象

- 個人塾・小規模塾のWebサイト
- フリーランス講師のHP/LP
- X/Twitter・InstagramなどSNSアカウント

## 評価基準（A/B/C/D）

| 判定 | 基準 |
|------|------|
| **A** | 即アプローチ価値あり。デザイン・情報設計共に優秀。 |
| **B** | 改善余地あり。一部問題あるが商談可能性あり。 |
| **C** | 現状では難しい。大規模改修が必要。 |
| **D** | 除外対象。連絡不可・業種不一致など。 |

## 必須出力フォーマット

以下の**固定形式**で出力すること。これにより次工程（nanami/itadori）への引き継ぎを短縮・高速化する。

```
## デザイン評価結果: [候補名]

**判定**: [A/B/C/D]
**対象URL**: [URL]
**評価日**: [YYYY-MM-DD]

### 判定理由3点

1. **[理由カテゴリ]**: [具体的な理由]
   - 根拠: [詳細説明・該当箇所]

2. **[理由カテゴリ]**: [具体的な理由]
   - 根拠: [詳細説明・該当箇所]

3. **[理由カテゴリ]**: [具体的な理由]
   - 根拠: [詳細説明・該当箇所]

### 即修正1点（今すぐ実施可能）

**提案**: [具体的な改善案]
**期待効果**: [数値・定性目標]
**工数**: [簡易/標準/大規模]

### 次アクション推奨

- A判定 → [即時アプローチ]
- B判定 → [条件付きアプローチ]
- C判定 → [保留・追加調査]
- D判定 → [除外・記録のみ]
```

## 理由カテゴリ一覧

判定理由は以下から**最大3つ**選択：

| カテゴリ | 説明 |
|----------|------|
| `VISUAL` | 視覚デザイン（配色・フォント・画像品質） |
| `LAYOUT` | レイアウト・構造（情報階層・視線誘導） |
| `UX` | ユーザー体験（導線・CTA・フォーム） |
| `CONTENT` | コンテンツ品質（文章・写真・情報量） |
| `BRANDING` | ブランディング（一貫性・差別化） |
| `TECH` | 技術面（表示速度・モバイル対応） |
| `TRUST` | 信頼性（実績表示・証明要素） |

## 即修正1点の選定基準

以下を優先して提案：

1. **低コスト・高インパクト**（1時間以内で実施可能）
2. **Edushiftの強みと関連**（HP改善・SNS運用・DX）
3. **具体性**（「〜しましょう」ではなく「この画像を差し替える」）

## 使用例

**入力**: 「早慶実践会 https://soukei-jissenkai.example.com を評価」

**出力**:
```
## デザイン評価結果: 早慶実践会

**判定**: B
**対象URL**: https://soukei-jissenkai.example.com
**評価日**: 2026-03-27

### 判定理由3点

1. **LAYOUT**: 情報階層が不明瞭
   - 根拠: トップページにサービス・講師紹介・受験実績が混在。導線が分散している

2. **VISUAL**: 写真品質にばらつき
   - 根拠: 講師写真はプロ仕様だが、教室・生徒写真がスマホ撮影品質

3. **CTA**: 問い合わせ導線が弱い
   - 根拠: 「お問い合わせ」ボタンがフッター近くにのみ配置。途中離脱を回収できていない

### 即修正1点（今すぐ実施可能）

**提案**: ファーストビューに「無料体験申込」ボタンを追加。スクロール追従型に変更。
**期待効果**: 問い合わせCV 20-30%向上
**工数**: 簡易（30分〜1時間）

### 次アクション推奨

B判定 → 条件付きアプローチ。HP改善提案を切り口にした営業が有効。
```

## 次工程引き継ぎ時の注意

- 本フォーマットを**そのまま**mailboxのbodyに含める
- Notion連携時は「即修正1点」を「次アクション」欄に転記
- 画像・スクリーンショットがあれば添付（optional）
